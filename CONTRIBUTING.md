# Contributing to WiseRecruiter

## Architecture overview

WiseRecruiter uses a **decoupled frontend** strategy: the .NET backend serves JSON APIs consumed by React components bundled with Vite. The Razor views are thin shells that mount React roots.

```
JobPortal/
  Controllers/AdminController.cs   ← JSON API actions (GetCandidatesJson, etc.)
  Models/ViewModels/               ← C# DTOs (source of truth for the JSON shape)
  ClientApp/
    src/types/                     ← TypeScript interfaces (must mirror C# DTOs)
    src/components/                ← React components + co-located tests
    vite.config.ts                 ← Vite build + Vitest config
  wwwroot/js/dist/                 ← Built JS bundles (git-ignored; generated by npm run build)
WiseRecruiter.Tests/
  Unit/DtoMatchesInterfaceTests.cs ← Contract test: C# ↔ TypeScript parity check
```

---

## The Golden Rule: keep C# DTOs and TypeScript interfaces in sync

The C# DTOs in `JobPortal/Models/ViewModels/` define the JSON contract.  
The TypeScript interfaces in `JobPortal/ClientApp/src/types/` must **exactly mirror** them.

> Any time you add, remove, or rename a property on a C# DTO you MUST:
> 1. Update the corresponding TypeScript interface in `src/types/`.
> 2. Run `npm run build` inside `ClientApp/` to regenerate the bundle.
> 3. Ensure `DtoMatchesInterfaceTests` passes (`dotnet test`).
> 4. Ensure the Vitest suite passes (`npm test`).

Failure to do this will be caught automatically by `DtoMatchesInterfaceTests.cs` during CI.

## Frontend development

### First-time setup
```bash
cd JobPortal/ClientApp
npm install
```

### Build the bundle (required before running the .NET app)
```bash
npm run build
# → outputs JobPortal/wwwroot/js/dist/candidate-dashboard.js
```

The .NET project's Debug build runs this automatically via an MSBuild target.  
In Release/CI builds, run `npm run build` explicitly before `dotnet publish`.

### Run Vitest (frontend unit tests)
```bash
npm test          # single run
npm run test:watch  # watch mode
```

### Hot-reload development
```bash
# Terminal 1 — .NET backend
dotnet run

# Terminal 2 — Vite dev server with proxy to localhost:5236
npm run dev
```

---

## Running the full test suite
```bash
# C# tests (includes the DTO contract check)
dotnet test

# Frontend tests
cd JobPortal/ClientApp && npm test
```

## Adding a new API endpoint + React component

1. Add a C# DTO in `JobPortal/Models/ViewModels/`.
2. Add a JSON action in `AdminController.cs` (or the relevant controller).
3. Add/update the TypeScript interface in `ClientApp/src/types/`.
4. Create the React component in `ClientApp/src/components/`.
5. Add a co-located `.test.tsx` with at minimum:
   - A render test (correct number of rows / nodes)
   - A state-dependent UI test (badges, icons, conditional rendering)
   - An interaction test (search, filter, click)
6. Add the mount point to the Razor `.cshtml` view and reference the bundle in a `@section Scripts` block.
7. Run `npm run build`, `npm test`, and `dotnet test` before committing.
